# 入出力仕様（Input/Output Contract）

## Python PY()セル → VBA `変換候補一覧を生成する()`

---

## 入力

### 入力1: 未変換材料リスト

**旧（Python）**: `xl(未変換一覧[], headers=False)` または fallback `xl(未一致一覧, headers=False)`

**新（VBA）**: `★変換済みデータ` シートを読み、`変換状態 = "未変換"` の行でフィルタ

| 列 | 内容 | 型 | 備考 |
|---|---|---|---|
| 区分 | "UR" または "通常" | text | `★変換済みデータ` の CONV_COL_区分 (5列目) |
| 材料 | 未変換の材料名（元の入力値） | text | `★変換済みデータ` の CONV_COL_材料 (6列目) |

処理前に (区分, 材料) のユニークペアに集約する（重複除外）。

**欠損処理**:
- 材料が空文字列 or null → スキップ
- 区分が空 → "通常" として扱う

### 入力2: 変換リスト

**旧（Python）**: `xl(テーブル1[], headers=False)`

**新（VBA）**: `テーブル1` テーブル（ListObject）を直接参照

| 列 | 内容 | 型 | 備考 |
|---|---|---|---|
| 変換前（材料名） | 照合元の材料名 | text | null行は除外 |
| 変換後（製品名） | 正式製品名 | text | null行は除外 |
| UR | UR対象品フラグ | text/null | "UR","○","1","TRUE","YES","対象" → UR品 |
| メーカー | メーカー名 | text | スコア計算には不使用 |

**欠損処理**:
- 変換前・変換後 どちらも null または空白 → 除外
- UR列 null → UR対象外として扱う

---

## 出力

**書き込み先**: `変換候補一覧` シート、行4（ヘッダ）、行5以降（データ）

| 列 | ヘッダ | 内容 | 型 |
|---|---|---|---|
| A | 元の材料名 | 元の未変換材料名 | text |
| B | 区分 | "UR" または "通常" | text |
| C | 候補1 | スコア最高の製品名 | text |
| D | スコア1 | 候補1の類似度スコア | number (小数1位) |
| E | 候補2 | スコア2位の製品名 | text |
| F | スコア2 | 候補2の類似度スコア | number (小数1位) |
| G | 候補3 | スコア3位の製品名 | text |
| H | スコア3 | 候補3の類似度スコア | number (小数1位) |

**注意**:
- 列C・E・Gは `LoadCandidateMap()`（Module2）が読み込む（変更禁止）
- 列I以降はユーザーが手動追記する拡張列（採用・UR・メーカー等）→ VBAは上書きしない
- 実行のたびに行5以降のA〜H列をクリアしてから再書き込みする

---

## VBA側の前提条件

実行前に以下が揃っていること:

1. `★変換済みデータ` シートが存在し、PQ更新済みであること
   - 列S（19列目）= "変換状態" ヘッダであること
   - `変換状態 = "未変換"` の行が存在すること
2. `テーブル1` テーブルが `変換リスト` シートに存在すること
3. `変換候補一覧` シートが存在すること
4. `プログラム設定` シート A3 が空または `FALSE` であること（`USE_PYTHON_PATH = False`）

---

## fallback動作の変更点

| 条件 | Python の挙動 | VBA の挙動 |
|---|---|---|
| 未変換データなし | 空DataFrameを出力 | 「未変換の材料はありません」メッセージで終了 |
| 変換リスト空 | スコア0の空候補を出力 | エラーメッセージで終了 |
| 区分列なし | `headers=False`で列名付与失敗 → except でフォールバック | `★変換済みデータ` から読むため区分は常に存在 |
