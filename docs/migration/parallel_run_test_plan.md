# 並走検証手順

## 前提

並走検証は「旧Python経路の出力スナップショット」と「新VBA経路の出力」を比較する。

PY()セルを再実行するにはPython環境が必要なため、**開発者PCで旧経路を先に実行してスナップショットを保存**する。

---

## ステップ1: 旧経路スナップショット取得（開発者PC、移行前）

```
1. Python in Excel が利用可能な環境で対象ブックを開く
2. PQを全更新（「データ」→「すべて更新」）
3. PY()セルが再計算されるまで待つ
4. 変換候補一覧 シート（A4:H最終行）をコピー
5. 新規ブックに「値のみ貼り付け」して保存
   → ファイル名: 変換候補一覧_旧経路スナップショット_YYYYMMDD.xlsx
6. このファイルを共有フォルダまたはリポジトリ外に保管
```

---

## ステップ2: VBA経路で候補生成

```
1. 同じブック（または同一データのコピー）で:
   プログラム設定 A3 = FALSE（またはブランク）に設定
2. PQを全更新
3. 「変換候補を生成」ボタンを押す（変換候補一覧を生成する() を実行）
4. 変換候補一覧 A4:H最終行をコピー
5. 新規ブックに「値のみ貼り付け」して保存
   → ファイル名: 変換候補一覧_新経路_YYYYMMDD.xlsx
```

---

## ステップ3: 差分比較

### 3-1. 件数確認

| チェック項目 | 合格基準 | 確認方法 |
|---|---|---|
| 行数（ユニーク材料数） | 旧経路 ≤ 新経路（新経路は未変換のすべてを含む） | 両ファイルの最終行番号を比較 |
| 旧経路で存在した材料が新経路で消えていない | 100% | VLOOKUP または COUNTIF で照合 |

### 3-2. 候補1一致率（主要確認軸）

```excel
# 新経路ファイルのI列に以下を入力:
=IFERROR(IF(C2=VLOOKUP(A2,旧経路!A:C,3,0),"一致","不一致"),"旧経路になし")
```

**合格基準**: 候補1の一致率 ≥ 80%

スコアの絶対値はアルゴリズムの差（SequenceMatcher vs bigram）で異なるが、**相対順位（候補1〜3の順番）の一致を優先して確認する**。

### 3-3. 空白率確認

| チェック項目 | 合格基準 |
|---|---|
| 候補1が空白の行 | 旧経路での空白率 ± 5%以内 |
| 候補1・2・3すべて空白の行 | 旧経路での発生件数以下 |

```excel
# 空白率計算
=COUNTBLANK(C:C)/COUNTA(A:A)
```

### 3-4. スコア傾向確認

```excel
# 候補1スコアの平均・最小・最大を比較
=AVERAGE(D5:D最終行)   ' 新経路
=AVERAGE(旧経路!D5:D最終行)   ' 旧経路
```

**合格基準**: 平均スコアの差が ±15pt 以内（bigram近似によるずれを許容）

---

## ステップ4: 判定

| 結果 | 対処 |
|---|---|
| すべての合格基準を満たす | 新VBA経路を本番切替（feature_flag_design.md のフェーズ2へ） |
| 候補1一致率が80%未満 | 不一致ケースを抽出し、正規化ロジックの差異を調査 |
| 件数が旧経路より少ない | ★変換済みデータの更新状態を確認。PQ更新漏れの可能性 |
| スコア傾向が大きく異なる | bigram類似度の閾値調整を検討（Module4 `SimilarityScore` 関数） |
